## Analysis & Hypothesis Formulation

### Architecture Design for Phase 1

With 16 binary pixels and kernel-3 stride-1 nodes:

- **Input embedding:** 16 binary scalars → 16 vectors of dim d (learnable 2×d lookup table for values 0 and 1)
- **Layer 1:** 14 nodes (positions 0–13), each reads 3 adjacent slots → 14 × d outputs
- **Layer 2:** 12 nodes → 12 × d outputs  
- **Layer 3:** 10 nodes → 10 × d outputs (top layer, used for linear probe)
- **Top-layer code:** Flatten 10 × d = 80-dim vector (with d=8) for classification

Each node: `f: R^{3d} → R^d` (encoder: linear map + nonlinearity; decoder for reconstruction loss; L1 sparsity on code)

### The Five Configurations

Interpreting P1-A through P1-E to create 5 maximally informative, distinct configurations:

| ID | Weight Sharing | d per slot | Key Question |
|---|---|---|---|
| P1-A | Within-layer only (3 weight matrices) | 8 | Standard CNN baseline: is within-layer sharing sufficient? |
| P1-B | Cross-layer (1 universal weight matrix) | 8 | **Core universal-node hypothesis** |
| P1-C | No sharing (~40 independent weight matrices) | 8 | Expressivity upper bound: what does sharing cost? |
| P1-D | Cross-layer (1 universal weight matrix) | 4 | Minimum viable recursion: does small d hurt? |
| P1-E | Cross-layer (1 weight matrix per d-level) | in=8, out=16 | Does breaking recursive dims help? |

### Training Protocol

- **No end-to-end backprop** across the hierarchy (per goal constraints)
- **Layer-by-layer local training:** each layer trained with local reconstruction loss (MSE + L1)
- For shared-weight configs (P1-B, P1-D, P1-E): simultaneous training of all layers with averaged local gradients updating the shared weights
- For P1-A: each layer trained independently with its own weights
- For P1-C: each node trained independently (most expensive, but provides the upper bound)
- Embedding layer trained jointly with Layer 1
- 100 epochs per training phase, batch size 32, lr=0.01

### Evaluation Protocol

- **Primary metric:** Linear-probe accuracy — train a simple logistic regression on top-layer codes to predict which of the 5 categories the input belongs to
- **Secondary metrics:** Parameter count comparison; per-layer reconstruction quality; code similarity analysis
- **Dataset:** Combined 5-category dataset (200 samples per category = 1000 train + 500 test)

### Critical Comparison

The make-or-break comparison is **P1-B vs P1-C**:
- If P1-B ≥ 80% AND P1-C - P1-B ≤ 15pp → universal node is viable
- If the gap exceeds 15pp → cross-layer weight sharing sacrifices too much
