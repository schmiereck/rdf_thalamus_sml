### Silver Lining: Weight Sharing Has Zero Cost

### The d=8 Bottleneck Question (per user hint)

The 3:1 compression ratio (24→8 per layer) is the same regardless of d. However, d=8 yields only 88 top-layer features for the linear probe. With d=16, there would be 176 features. For reconstruction, the bottleneck forces the encoder to allocate all 8 dimensions to reconstruction fidelity, leaving zero capacity for category structure. For **latent-space objectives like JEPA**, the bottleneck is less critical because the prediction target is itself a learned representation — the encoder can choose what information to preserve.

### Strategic Decision: Stay in Phase 1, Fix the Objective

The user's recommendation is correct. Moving to Phase 2 (temporal) before solving the objective problem would compound failures. If JEPA works spatially, it also validates temporal applicability (same kernel-3 structure, same objective, different axis). Two phases in one.

## Hypothesis Formulation

**Core Mechanism:** The reconstruction objective fails because it requires preserving all pixel-level information within the 3:1 bottleneck, forcing the encoder to sacrifice category structure. Latent-space objectives (JEPA, contrastive, SFA) avoid this misalignment by operating in the code space where the encoder controls what information to preserve.

**Prediction:** A local JEPA objective — predicting the latent code of a spatially adjacent position at each layer — will force the encoder to learn representations where category-relevant features are predictable across neighbors. This should achieve ≥60% accuracy because:
1. Category boundaries create predictable spatial patterns (e.g., blobs have smooth spatial structure)
2. Predicting in latent space avoids reconstruction misalignment
3. The architecture already produces 48% accuracy with random weights — any objective that preserves this structure while adding category relevance should exceed it

**Falsification:** If none of the four alternative objectives (JEPA, contrastive, SFA, Hebbian) exceeds the untrained baseline of 48.4% at d=8, then the hypothesis that local objectives can improve on random projections for this architecture is falsified. Specifically, if JEPA < 50%, the JEPA-specific claim is falsified.

### Detailed Implementation Specifications

**P1-F: Local JEPA**
- Encoder: Same P1-B (3 layers, kernel-3, stride-1, d=8, cross-layer shared weights)
- Predictor per layer: Linear(d → d) — shared across positions within a layer, NOT shared across layers
- Loss per layer l: `Σ_i [‖pred(z_l[i]) − sg(z_l[i+1])‖² + ‖pred(z_l[i+1]) − sg(z_l[i])‖²]`
- VICReg collapse prevention:
  - Variance: `Σ_d max(0, γ − std(z_d))²` where γ=1.0
  - Covariance: off-diagonal penalty on covariance matrix of z
- Total loss: `Σ_l pred_loss_l + λ_var·var_loss + λ_cov·cov_loss`
- λ_var=25, λ_cov=1 (VICReg defaults, tuned if needed)

**P1-G: Contrastive**
- Augmentation: Randomly flip 1-2 bits out of 16 (spatial corruption)
- Encode both views → flatten top-layer codes → L2 normalize
- InfoNCE loss with temperature τ=0.5
- Batch size ≥ 32 for sufficient negatives
- Projection head: MLP(88 → 44 → 22) for contrastive loss only (discarded at eval)

**P1-H: Slow Feature Analysis**
- At each layer: `SFA_loss_l = (1/N) Σ_i ‖z_l[i] − z_l[i−1]‖²`
- Variance constraint: `var_loss = Σ_d (Var(z_d) − 1)²`
- Total loss: `Σ_l sfa_loss_l + λ_var·var_loss`
- λ_var=10

**P1-I: Hebbian + Decorrelation**
- Hebbian loss per node: `H = −(1/N) Σ y·x` (maximize correlation)
- Oja normalization: `ΔW = η·y·(x − y·W)` → equivalent to weight decay proportional to output
- Decorrelation: minimize off-diagonal of covariance matrix
- Total loss: `−Σ H + λ_oja·‖W‖² + λ_deco·off_diag(Cov(z))`
- Implemented as gradient descent on a differentiable loss (not online Oja update) for fair comparison
