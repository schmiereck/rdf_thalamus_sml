## Phase 3 Architecture Design

Based on the UniversalNode (kernel-3, stride-1, d=16) that works on both spatial (Phase 1) and temporal (Phase 2) data:

**Input**: (T=32, S=16) binary matrix — 32 timesteps of 16-bit spatial patterns

**Processing model**: Each "grid layer" consists of a spatial pass followed by a temporal pass:
- Spatial pass: At each time step, apply UniversalNode along S axis (kernel-3, stride-1)
- Temporal pass: At each spatial position, apply UniversalNode along T axis (kernel-3, stride-1)

With L=3 alternating passes each axis:
- Start: (32, 16, 1)
- After 3 spatial + 3 temporal passes: (26, 10, d=16) → 4,160-dim top-layer code

**Three architectures**:
- **P3-A**: Spatial encoder (3 layers, spatial-only weights) → Temporal encoder (3 layers, temporal-only weights). Trained sequentially.
- **P3-B**: Alternating spatial/temporal passes with separate W_spatial and W_temporal. Trained jointly.
- **P3-C**: Same alternating passes but W_spatial = W_temporal = W_shared. Single weight set. Trained jointly. This is the strong universal hypothesis.

**Key parameter comparison**: P3-C uses ~half the node parameters of P3-B/A, testing whether the shared-weight constraint costs expressivity.

**JEPA training objective**: For each node output z at (s,t), predict neighbors along the applied axis (bidirectional), with VICReg collapse prevention. For P3-C, spatial and temporal JEPA losses both update the shared W.

**Four benchmark tasks** (as specified in goal.md):
1. Moving blob: blob translating across space over time (direction classification)
2. Expanding/contracting blob: blob changing size (growth pattern classification)
3. Periodic spatiotemporal patterns: regularity in both axes (period classification)
4. Object permanence: blob disappears for k steps then reappears (gap classification)

**Evaluation**: Linear probe on top-layer codes, 4-5 class classification per task + overall aggregate.

**Critical metric**: gain-over-untrained (to control the periodicity loophole from Phase 2).

## Hypothesis Formulation

The central question from Phase 2: zero-shot weight transfer failed, but can *joint* training discover a representation compatible with both axes?

**Argument for P3-C viability**: Joint training simultaneously optimizes for spatial and temporal prediction, potentially finding a representation subspace where both objectives are satisfied. The UniversalNode's kernel-3 sliding window has identical structure along both axes — the only difference is the data statistics. Joint training could align these statistics.

**Argument against**: Phase 2 showed that spatially-optimal weights are useless temporally. This suggests the two axes require fundamentally different projections. Even with joint training, a single weight matrix may be forced to compromise, losing expressivity on both axes.

**Hypothesis**: Joint spatiotemporal JEPA training enables a single UniversalNode weight set (P3-C) to learn representations where spatial and temporal prediction are compatible, achieving at least 80% of P3-A's accuracy on spatiotemporal pattern recognition (gap ≤ 20 percentage points).

**Why this might work despite Phase 2**: Zero-shot transfer fails because spatial weights are *overfitted* to spatial structure. Joint training from scratch doesn't have this problem — it never commits to one axis's statistics at the expense of the other. The JEPA objective provides a learning signal along both axes simultaneously.

**Falsification**: If P3-C's aggregate accuracy is more than 20pp below P3-A, the shared-weight constraint is too restrictive for practical use.
